# Repository Migration Steps

## Step 1: Create Private Repository on GitHub

1. Go to https://github.com/new
2. Repository name: `Mailchimp-MCP-by-Alien-Lifestyles` (or `mailchimp-mcp-by-alien-lifestyles`)
3. Description: "Mailchimp MCP Server - Complete GTM repository"
4. Set to **Private**
5. Do NOT initialize with README, .gitignore, or license
6. Click "Create repository"

## Step 2: Push to New Private Repository

After creating the repository, GitHub will show you the repository URL. Use that URL in the commands below.

## Step 3: Rename Original Repository

1. Go to https://github.com/alien-lifestyles/mailchimp-mcp-alien-lifestyles/settings
2. Scroll to "Repository name" section
3. Change name to: `MailchimpMCP_v0`
4. Click "Rename"
5. Repository remains fully active (no archiving)

## Verification

After migration, verify:
- [ ] New private repo has all branches
- [ ] New private repo has all tags
- [ ] New private repo has all commits
- [ ] Original repo renamed successfully
- [ ] Can clone new private repo
